## オーバーロードを理解しよう (Method Overloading) C# ↔ Java 비교

### 요약
> **오버로드(Overload) = 같은 이름의 메서드를 매개변수(파라미터)만 다르게 하여 여러 개 정의하는 것.**
> - 반환형(return type)은 달라도 인정 X
> - 반드시 매개변수의 개수/타입이 달라야 함
> - 호출 시 컴파일러가 “어떤 시그니처를 쓸지” 자동으로 선택
>   
> **C#과 Java 모두 동일한 개념.**

### C# 코드
```
using System;

class Lesson08
{
    public static void Main()
    {
        Attack();  
        Attack("スライム");
        Attack(10);
        Attack("ドラゴン", 3);
    }
    
    // 매개변수 없는 버전
    static void Attack()
    {
        Console.WriteLine("勇者は、敵を攻撃した。");
    }

    // 문자열 버전
    static void Attack(string target)
    {
        Console.WriteLine("勇者は、" + target + "を攻撃した。");
    }

    // 숫자 버전
    static void Attack(int number)
    {
        Console.WriteLine("勇者は、" + number + "匹を攻撃した。");
    }

    // 문자열 + 숫자 버전
    static void Attack(string target, int number)
    {
        Console.WriteLine("勇者は、" + number + "匹の" + target + "を攻撃した。");
    }
}

```

### 실행 결과
```
勇者は、敵を攻撃した。
勇者は、スライムを攻撃した。
勇者は、10匹を攻撃した。
勇者は、3匹のドラゴンを攻撃した。
```

---

### 왜 이런 결과가 나올까? (오버로드 설명)
> 컴파일러는 호출된 메서드를 이렇게 선택함 :

| 호출                  | 호출되는 시그니처             |
| ------------------- | --------------------- |
| `Attack()`          | `Attack()`            |
| `Attack("スライム")`    | `Attack(string)`      |
| `Attack(10)`        | `Attack(int)`         |
| `Attack("ドラゴン", 3)` | `Attack(string, int)` |

> **즉, 같은 이름이지만 파라미터 조합이 다르기 때문에 서로 다른 메서드로 취급된다.**

---

### Java에서도 동일한 코드 가능

```
public class Lesson08 {
    public static void main(String[] args) {
        attack();
        attack("スライム");
        attack(10);
        attack("ドラゴン", 3);
    }

    static void attack() {
        System.out.println("勇者は、敵を攻撃した。");
    }

    static void attack(String target) {
        System.out.println("勇者は、" + target + "を攻撃した。");
    }

    static void attack(int number) {
        System.out.println("勇者は、" + number + "匹を攻撃した。");
    }

    static void attack(String target, int number) {
        System.out.println("勇者は、" + number + "匹の" + target + "を攻撃した。");
    }
}

```

---

### C# ↔ Java 비교표

| 개념              | C#              | Java     | 비고             |
| --------------- | --------------- | -------- | -------------- |
| 오버로드 문법         | 동일              | 동일       | 매개변수 조합만 다르면 됨 |
| 반환형만 변경         |  오류            |  오류     | 시그니처가 구분되지 않음  |
| 오버로드 선택 방식      | 컴파일 시 결정        | 컴파일 시 결정 | 런타임 X          |
| static/instance | 둘 다 Overload 가능 | 동일       | 제한 없음          |
