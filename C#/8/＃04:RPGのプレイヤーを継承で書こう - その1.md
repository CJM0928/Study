## RPGのプレイヤーを継承で書こう – その1 (C# ↔ Java)

### 요약

- Player 클래스를 여러 번 생성해 파티 배열을 구성
- foreach 루프로 모든 플레이어가 같은 적을 공격
- 아직 상속은 등장하지 않고, "파티 관리 로직"을 이해하는 단계

---

### C# 코드 (해설 포함)

```
using System;

class Lesson08
{
    public static void Main()
    {
        Console.WriteLine("=== パーティーでスライムと戦う ===");

        var hero = new Player("勇者");
        var warrior = new Player("戦士");

        // 파티 배열
        Player[] party = { hero, warrior };

        // 모든 파티원이 슬라임을 공격
        foreach (var player in party)
        {
            player.Attack("スライム");
        }
    }
}

class Player
{
    public string Name { get; private set; }
    
    // 생성자(Constructor)
    public Player(string name)
    {
        // 전달받은 매개변수 name을
        // 이 객체의 Name 프로퍼티(필드)에 저장하는 역할
        // 즉, 객체 생성 시 이름이 설정된다.
        Name = name;
    }
    
    public void Attack(string enemy)
    {
        Console.WriteLine(Name + "は、" + enemy + "を攻撃した！");
    }
}

```

### 출력 결과
```
=== パーティーでスライムと戦う ===
勇者は、スライムを攻撃した！
戦士は、スライムを攻撃した！
```

---

### 생성자 구간 설명 (주석 상세 버전)

```
public Player(string name)
{
    // (매개변수 name) 예: "勇者"
    // ↓
    // Player 객체의 Name 프로퍼티에 저장한다.
    // 결국 hero.Name == "勇者" 가 된다.
    Name = name;
}
```

### 왜 필요한가?
- 객체마다 서로 다른 이름을 가지기 위해
- Player를 설계할 때 Name을 외부에서 넣는 방식으로 만들었기 때문
- 이게 바로 객체마다 상태(state)가 다르다는 OOP의 핵심

---

### Java로 동일 기능 구현

```
class Player {
    private String name;

    public Player(String name) {
        this.name = name; // Java도 동일한 역할
    }

    public void attack(String enemy) {
        System.out.println(name + "は、" + enemy + "を攻撃した！");
    }
}

public class Lesson08 {
    public static void main(String[] args) {
        System.out.println("=== パーティーでスライムと戦う ===");

        Player hero = new Player("勇者");
        Player warrior = new Player("戦士");

        Player[] party = { hero, warrior };

        for (Player p : party) {
            p.attack("スライム");
        }
    }
}
```

---

### C# vs Java 비교표

| 항목      | C#                        | Java                    | 설명        |
| ------- | ------------------------- | ----------------------- | --------- |
| 인스턴스 생성 | `new Player("勇者")`        | 동일                      | 생성자 호출    |
| 필드 저장   | `Name = name`             | `this.name = name`      | 개념 동일     |
| 배열 루프   | `foreach(var p in party)` | `for(Player p : party)` | 문법만 다름    |
| 출력      | `Console.WriteLine()`     | `System.out.println()`  | 네임스페이스 차이 |
