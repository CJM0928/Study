## 名前空間(Namespace) & 標準ライブラリ — C# ↔ Java

### 1. Namespace란?
스크린샷 내용 요약:
- System 이름공간 안에 `Array`, `Console`, `Exception`, `Math`, `Random` … 같은 클래스들이 묶여 있음
- System.IO 이름공간 안에 `File`, `FileInfo`, `FileStream`, `Directory`, `Path` … 같은 입출력 관련 클래스들이 묶여 있음
- 즉, 관련된 클래스들을 “폴더”처럼 그룹화한 것이 바로 Namespace.

#### C#에서의 구조

```
System.Console.WriteLine("Hello World");
^-----^  ^-----^  ^----------^
  네임스페이스   클래스        메서드
```

#### using 키워드

```
using System;

Console.WriteLine("Hello World");
// System. 을 생략 가능
```

⇒ using System; 을 적으면 같은 파일 안에서는 System.을 생략하고
Console.WriteLine 만 써도 된다.

---

### 2. Java에서의 대응 개념 (package + import)
> C#의 namespace + using ↔ Java의 package + import

```
import java.util.List;
import java.util.ArrayList;

List<String> list = new ArrayList<>();
// java.util. 을 매번 쓰지 않아도 됨
```

---

| 개념    | C#                              | Java                           | 비고                   |
| ----- | ------------------------------- | ------------------------------ | -------------------- |
| 묶음 이름 | namespace `System`, `System.IO` | package `java.util`, `java.io` | 역할 동일                |
| 사용 선언 | `using System;`                 | `import java.util.*;`          | prefix 생략            |
| 전체 이름 | `System.Console.WriteLine`      | `java.util.ArrayList`          | fully qualified name |

### 3. 標準ライブラリ List 사용 예제 (C#)

```
// 標準ライブラリを使ってみよう
using System;                     // Console, String 등
using System.Collections.Generic; // List<T> 가 들어있는 namespace

class Lesson08
{
    public static void Main()
    {
        // List<string> = 문자열을 담는 동적 배열
        var team = new List<string>();

        // 뒤에 추가
        team.Add("勇者");
        team.Add("戦士");
        team.Add("魔法使い");

        // ① 현재 파티 출력
        foreach (var member in team)
        {
            Console.WriteLine(member);
        }

        Console.WriteLine("==========");

        // ② 인덱스 2 위치에 "忍者" 삽입 (0:勇者,1:戦士,2:忍者,...)
        team.Insert(2, "忍者");
        foreach (var member in team)
        {
            Console.WriteLine(member);
        }

        Console.WriteLine("==========");

        // ③ 값이 "忍者" 인 요소 제거 (처음 발견되는 하나)
        team.Remove("忍者");
        foreach (var member in team)
        {
            Console.WriteLine(member);
        }
    }
}
```

```

魔法使い

忍者
魔法使い

勇者
```

### 4. Java 버전으로 쓰면?
```
import java.util.ArrayList;
import java.util.List;

public class Lesson08 {
    public static void main(String[] args) {
        // List<String> = 인터페이스, ArrayList = 구현체
        List<String> team = new ArrayList<>();

        team.add("勇者");
        team.add("戦士");
        team.add("魔法使い");

        for (String member : team) {
            System.out.println(member);
        }

        System.out.println("==========");

        // 인덱스 2에 삽입
        team.add(2, "忍者");
        for (String member : team) {
            System.out.println(member);
        }

        System.out.println("==========");

        // 값으로 삭제
        team.remove("忍者");
        for (String member : team) {
            System.out.println(member);
        }
    }
}
```

### 5. List 조작 메서드 비교표

| 기능      | C# `List<T>`              | Java `List<E>`      | 설명            |
| ------- | ------------------------- | ------------------- | ------------- |
| 뒤에 추가   | `Add(item)`               | `add(item)`         | 맨 끝에 요소 추가    |
| 중간 삽입   | `Insert(index, item)`     | `add(index, item)`  | 지정 인덱스에 삽입    |
| 값으로 삭제  | `Remove(item)`            | `remove(Object o)`  | 첫 번째 일치 요소 제거 |
| 인덱스로 삭제 | `RemoveAt(index)`         | `remove(int index)` | 해당 위치 요소 제거   |
| 반복      | `foreach (var x in list)` | `for (E x : list)`  | 향상된 for 문     |

--- 

### 요약

- Namespace = 클래스들을 폴더처럼 묶는 단위
- `System.Console.WriteLine` 에서
- `System` = 네임스페이스, `Console` = 클래스, `WriteLine` = 메서드
- `using System;` 을 쓰면 `System.` 생략 가능
- 표준 라이브러리의 List<T> 는 Java의 List<E> / ArrayList<E> 와 거의 1:1 대응
- Add / Insert / Remove 로 자유롭게 파티 멤버를 조작할 수 있다
