## クラスとオブジェクト (C# ↔ Java)

### 오브젝트
- 객체(Object)는 클래스(Class)라는 설계도로부터 만들어지는 데이터(변수) + 행동(메서드)의 묶음이다.
- 변수(フィールド) → 객체의 상태(state)
- 메서드(メソッド) → 객체의 행동(behavior)
  
**같은 클래스라도 각 객체는 서로 다른 변수를 가질 수 있지만, 메서드는 공유된 형태를 사용한다**

---

### 요약
- 클래스(Class): 객체의 설계도 (필드 + 메서드 정의)
- 객체(Object): 클래스로부터 생성된 실체
- 상속(Inheritance): 기존 클래스를 기반으로 새로운 클래스를 정의

---

### 비교 요약표
  | 概念        | C#                     | Java                         | 備考                   |
| --------- | ---------------------- | ---------------------------- | -------------------- |
| クラス定義     | `class Hero { ... }`   | `class Hero { ... }`         | 문법 거의 동일             |
| インスタンス生成  | `var h = new Hero();`  | `Hero h = new Hero();`       | C#은 `var` 사용 가능      |
| メンバ呼び出し   | `h.Attack();`          | `h.attack();`                | 메서드 명명 규칙만 다름        |
| 継承        | `class Mage : Hero {}` | `class Mage extends Hero {}` | 콜론(`:`) vs `extends` |
| 基底コンストラクタ | `base()`               | `super()`                    | 부모 클래스 생성자 호출        |
| オーバーライド   | `override` 키워드         | `@Override` 어노테이션            | 둘 다 명시 필요            |

---

### 핵심 차이 요약
| 항목            | C#                      | Java                 |
| ------------- | ----------------------- | -------------------- |
| 접근 제한자 기본값    | `private`               | package-private      |
| `override` 명시 | 필수 (`override`)         | 권장 (`@Override`)     |
| `base` 키워드    | 부모 접근 (`base.Method()`) | `super`              |
| 생성자 호출 순서     | 부모 → 자식                 | 부모 → 자식              |
| 다중상속          | 불가 (`interface`로 대체)    | 불가 (`interface`로 대체) |
