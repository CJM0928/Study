## メソッドのオーバーライド (C# ↔ Java)

### 요약
- 부모 클래스의 메서드를 자식 클래스에서 재정의(override) 하는 기능
- C#: 부모는 virtual, 자식은 override
- Java: 부모/자식 둘 다 @Override 사용 (virtual 개념은 기본)

---

### C# 코드 — 메서드 오버라이드 예제
```
using System;

class Lesson08
{
    public static void Main()
    {
        var box = new Box("鋼鉄の剣");
        box.Open();
        
        var magicBox = new MagicBox("モノマネモンスター");
        magicBox.Look();
        magicBox.Open();
    }
}

class Box
{
    public string Item { get; private set; }

    public Box(string item)
    {
        Item = item;
    }

    public virtual void Open()
    {
        Console.WriteLine("宝箱を開いた。" + Item + "を手に入れた。");
    }
}

class MagicBox : Box
{
    public MagicBox(string item) : base(item) {}

    public void Look()
    {
        Console.WriteLine("宝箱は妖しく輝いている。");
    }
    
    public override void Open()
    {
        Console.WriteLine("宝箱を開いた。" + Item + "が襲ってきた！");
    }
}

```

### 실행 결과
```
宝箱を開いた。鋼鉄の剣を手に入れた。
宝箱は妖しく輝いている。
宝箱を開いた。モノマネモンスターが襲ってきた！
```

### Java 버전 (동일 구조)
```
class Box {
    protected String item;

    public Box(String item) {
        this.item = item;
    }

    public void open() {
        System.out.println("宝箱を開いた。" + item + "を手に入れた。");
    }
}

class MagicBox extends Box {

    public MagicBox(String item) {
        super(item);
    }

    public void look() {
        System.out.println("宝箱は妖しく輝いている。");
    }

    @Override
    public void open() {
        System.out.println("宝箱を開いた。" + item + "が襲ってきた！");
    }
}

public class Lesson08 {
    public static void main(String[] args) {
        Box box = new Box("鋼鉄の剣");
        box.open();

        MagicBox magicBox = new MagicBox("モノマネモンスター");
        magicBox.look();
        magicBox.open();
    }
}

```

### C# vs Java 오버라이드 비교표

| 기능            | C#                            | Java                    | 포인트                           |
| ------------- | ----------------------------- | ----------------------- | ----------------------------- |
| 부모 메서드 재정의 허용 | `virtual`                     | 기본적으로 가능                | C#만 virtual 필요                |
| 자식에서 오버라이드    | `override`                    | `@Override`             | 둘 다 명시하는 것을 권장                |
| 부모 생성자 호출     | `base(item)`                  | `super(item)`           | 동일한 역할                        |
| 필드 접근         | `public string Item { get; }` | `protected String item` | Java는 getter를 따로 만들어야 하는 경우 多 |
| 메서드 실행 결과     | 자식 override 메서드가 실행됨          | 동일                      | 다형성(polymorphism)             |
