## 簡単な例外処理をしてみよう (C# ↔ Java)

### 예외 처리 기본 구조

```
try {
    // 예외가 발생할 수 있는 코드
}
catch (Exception e) {
    // 예외 발생 시 실행되는 코드
}
finally {
    // 예외 발생 여부와 상관없이 반드시 실행되는 코드
}

```

> try → 위험한 코드를 넣는다  
> catch → 예외를 잡는다  
> finally → 항상 실행되는 구간 (리소스 정리용)

### C# 코드 (해설 주석 포함)

```
using System;

class Lesson10
{
    public static void Main()
    {
        Console.WriteLine("Hello World");
       
        try
        {
            int number = 0;
            int answer = 100 / number;   // ← 여기서 예외 발생
            Console.WriteLine(answer);   // 실행되지 않음
        } 
        catch (Exception e)
        {
            // 예외 객체 e의 메시지를 출력
            Console.WriteLine(e.Message);
        }
        finally
        {
            // 예외 발생 여부에 상관없이 반드시 실행됨
            Console.WriteLine("Hello C#");
        }
    }
}
```

---

### 실행 결과

```
Hello World
Attempted to divide by zero.
Hello C#
```

---

### 흐름 설명

- Hello World 출력
- 100 / 0 에서 DivideByZeroException 발생
- catch (Exception e) 로 이동 → 메시지 출력
- finally 실행 → "Hello C#" 출력
- 프로그램 정상 종료

---

### Exception e 의 의미
> Exception e 는 예외 정보를 담은 객체

- e.Message → 예외 메시지
- e.StackTrace → 호출 스택 정보
- e.GetType() → 예외 타입

--- 

### Java에서는 동일한 코드

```
public class Lesson10 {
    public static void main(String[] args) {
        System.out.println("Hello World");

        try {
            int number = 0;
            int answer = 100 / number;  // ArithmeticException
            System.out.println(answer);
        } catch (Exception e) {
            System.out.println(e.getMessage());
        } finally {
            System.out.println("Hello Java");
        }
    }
}
```

---

### 출력결과
```
Hello World
/ by zero
Hello Java
```

### C# ↔ Java 예외 처리 비교

| 요소        | C#                      | Java                  | 설명           |
| --------- | ----------------------- | --------------------- | ------------ |
| try/catch | 동일                      | 동일                    | 구조 같음        |
| finally   | 동일                      | 동일                    | 항상 실행        |
| 예외 메시지    | `e.Message`             | `e.getMessage()`      | 메서드 이름 차이    |
| 예외 타입     | `DivideByZeroException` | `ArithmeticException` | 언어별 기본 예외 이름 |
