## 例外処理の概要 (C# ↔ Java)

### 프로그램 실행 흐름 (Compilation → Execution)

```
ソースコード
    ↓
コンパイル（Compile）
    ↓
実行（Run / Execute）
```

### 1) コンパイル（Compile） 단계
#### 코드가 실행 가능한 형태로 변환되는 과정
> C# 컴파일 에러 예시

- 문법 오류
- 세미콜론 누락
- 타입 불일치
- 존재하지 않는 메서드 호출

➡ 실행 전에 에러가 발생하기 때문에 프로그램이 시작조차 되지 않음

### 2) 実行（Run） 단계
> ※ 여기서 프로그램이 돌아가는 도중에 문제가 발생하면 例外(Exception) 이 발생
> 이미지의 빨간 원: **実行時エラー（Runtime Error）**

예:
- 계산 불가능
- 파일이 존재하지 않음
- 네트워크 연결 실패
- 배열 범위 벗어남

➡ 프로그램이 강제 종료될 수 있다 → 그래서 “예외 처리(Exception Handling)”가 필요함.

---

### 例外が発生する例（C# ↔ Java 동일）

| 예외 상황            | 설명                                      |
| ---------------- | --------------------------------------- |
| ゼロで割り算           | 0으로 나누기 → `DivideByZeroException`       |
| 数値変換で、数字でない文字を指定 | `"abc"` 를 숫자로 변환 시도 → `FormatException` |
| 配列の範囲外にアクセス      | 인덱스 초과 → `IndexOutOfRangeException`     |
| ファイルが存在しない       | 경로가 잘못됨 → `FileNotFoundException`      |

---

### C# 예외 발생 예시

```
int a = 10;
int b = 0;
int result = a / b;  // DivideByZeroException
```

```
int num = int.Parse("abc"); // FormatException
```

```
int[] arr = {1, 2, 3};
Console.WriteLine(arr[5]); // IndexOutOfRangeException
```

```
File.ReadAllText("missing.txt"); // FileNotFoundException
```

---

### Java에서는 이렇게 됨

```
int a = 10;
int b = 0;
int result = a / b;  // ArithmeticException
```

```
int num = Integer.parseInt("abc"); // NumberFormatException
```

```
int[] arr = {1, 2, 3};
System.out.println(arr[5]); // ArrayIndexOutOfBoundsException
```

```
Files.readString(Path.of("missing.txt")); // NoSuchFileException
```

---

### C# ↔ Java 예외 비교표

| 예외 상황     | C# 예외                      | Java 예외                          | 비고          |
| --------- | -------------------------- | -------------------------------- | ----------- |
| 0 나누기     | `DivideByZeroException`    | `ArithmeticException`            | 둘 다 런타임     |
| 잘못된 숫자 변환 | `FormatException`          | `NumberFormatException`          | 기본 자료형 변환 시 |
| 배열 범위 초과  | `IndexOutOfRangeException` | `ArrayIndexOutOfBoundsException` | 이름만 다름      |
| 파일 없음     | `FileNotFoundException`    | `NoSuchFileException`            | I/O 관련      |
